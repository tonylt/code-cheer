Append a new `test-node` job to the existing `.github/workflows/ci.yml` file, after the existing `test` job. The existing `test` job (Python matrix) must remain completely unchanged.

Add this job at the end of the `jobs:` section (per D-05, D-06, D-07):

```yaml
  test-node:
    runs-on: ubuntu-latest
    strategy:
      fail-fast: false
      matrix:
        node-version: ["20", "22"]

    steps:
      - uses: actions/checkout@v4

      - name: Set up Node.js ${{ matrix.node-version }}
        uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node-version }}
          cache: "npm"

      - name: Install dependencies
        run: npm ci

      - name: Build
        run: npm run build

      - name: Type check
        run: npm run typecheck

      - name: Validate cold start <100ms
        run: |
          START=$(date +%s%3N)
          node dist/statusline.js
          END=$(date +%s%3N)
          MS=$((END - START))
          echo "Cold start: ${MS}ms"
          if [ "$MS" -gt 100 ]; then
            echo "FAIL: cold start ${MS}ms exceeds 100ms threshold"
            exit 1
          fi
          echo "PASS: cold start ${MS}ms"
```

Key details:
- `fail-fast: false` (per D-06) — Node 20 failure does not cancel Node 22
- `cache: "npm"` uses actions/setup-node@v4 built-in npm caching (requires package-lock.json in repo)
- Build and Type Check are separate steps (per D-03) — each has independent error output
- Cold start validation uses `date +%s%3N` for millisecond timing and exits 1 if >100ms

Do NOT modify the existing `test` job in any way. Only append the new `test-node` job.

Note per D-08: Branch protection rules need manual GitHub UI configuration to add `test-node (20)` and `test-node (22)` as required status checks. This is a manual step documented in 08-VALIDATION.md, not automated here.
