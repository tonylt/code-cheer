## Accepted Scope (added to this plan)

### core/git_context.py
- `load_git_context(cwd)` 接收 workspace 路径（从 stdin JSON 读取），所有 subprocess 以 cwd 运行
- **3 个并行 subprocess**（concurrent.futures.ThreadPoolExecutor, timeout=5s/个）：
  1. `git log --oneline --since=midnight` → commits_today（同时用于 milestone 检测）
  2. `git diff --stat HEAD` → diff_lines（big_diff 事件）
  3. `git log --format="%ai" --since=midnight` → 最早提交时间戳（用于 session_minutes 辅助计算）
  - 错误策略：某个 subprocess 超时/失败时 fallback 到 0，不阻塞整体（意味着该轮 big_diff 不触发，属于可接受的降级）
  - 实际耗时：并行后总时间由最慢的 subprocess 决定，本地 repo 通常 50-200ms；非 320ms 串行

### state.json 新增字段
- `session_start`: 首次 `statusline.py --update` 调用时若缺失则设为当前时间戳；用于计算 `session_minutes = (now - session_start) / 60`
- `commits_today`: 由 --update 写入，render 模式读取（无 subprocess）
- `last_git_events`: 数组，使用细粒度 key——`milestone_5`/`milestone_10`/`milestone_20` 独立存储，防止过 5 次后抑制 10/20 次触发

### config.json 新增字段
```json
{
  "character": "nova",
  "event_thresholds": {
    "big_diff": 200,
    "milestone_counts": [5, 10, 20],
    "big_session_minutes": 120,
    "long_day_commits": 15,
    "late_night_hour_start": 22
  }
}
```
trigger.py 读取 config 阈值；字段缺失时回落到上述默认值。`late_night_hour_start` 使用本地时间（不转换 UTC）。

### state.json 容错规则
- `session_start` 缺失或无效（非 ISO timestamp）→ 重置为 `now()`
- `last_git_events` 非 list → 重置为 `[]`
- `commits_today` 非 int → 重置为 `0`
- 任何字段读取异常时静默使用默认值，不抛出异常影响 render

### 重命名
- `late_night_push` → `late_night_commit`（触发点是 commit 而非 push；push 无法从 git log 低成本感知）

### --debug-events 模式
```
$ python3 statusline.py --debug-events
[2026-04-01T23:15:00] GIT_CONTEXT: {"commits_today": 6, "diff_lines": 312, "session_minutes": 47}
EVENTS_WOULD_FIRE: {"big_diff": "diff_lines 312 >= threshold 200", "late_night_commit": "hour 23"}
STATE_SNAPSHOT: {"last_git_events": ["big_diff"], "commits_today": 6, ...}
```
输出到 stderr，不影响 render 模式正常输出。

### 成功标准（修订）
- 用户提交后，**下一次 statusline.py --update 运行时**（即下一次 Claude 响应结束时）状态栏显示对应角色消息
- 不需要额外触发器；Stop hook 已在每次 Claude 响应后自动运行
